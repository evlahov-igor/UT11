Операции закрытия месяца

Рабочее место предназначено для последовательного выполнения регламентных операций закрытия месяца, диагностики состояния каждой операции, а также настройки автоматического выполнения операций по расписанию.

Закрытие месяца может выполняться как по одной, так и по нескольким организациям, доступным текущему пользователю. При этом одновременное закрытие месяца по одной и той же организации в нескольких сеансах не допускается - при попытке запустить закрытие месяца по организации, по которой уже выполняется закрытие месяца в другом сеансе, будет выдано сообщение об ошибке. Информацию об активных сеансах закрытия месяца можно получить с помощью соответствующей команды в подменю Еще.

Предусмотрено два режима закрытия месяца:
- Предварительное закрытие
- Окончательное закрытие

Режим Предварительное закрытие используется для расчета предварительной себестоимости номенклатуры по методу оценки стоимости, указанному в учетной политике.
Для предварительного расчета указывается дата месяца, по которую необходимо выполнить расчет. Расчет будет выполнен за период с начала месяца по указанную дату включительно.
Документы с датой, позднее даты предварительного закрытия месяца, не будут влиять на результат расчета себестоимости.

Режим Окончательное закрытие используется для выполнения всех регламентных операций за указанный месяц.

На форме присутствуют этапы выполняемые вручную и выполняемые автоматически. Ручные этапы показываются только в случае если их необходимо выполнить, в противном случае - они не отображаются. Все ручные операции находятся в самой первой группе операций.

Выполнять ручные этапы необходимо перед выполнением автоматических. Запуск вручную автоматической операции ведет к пересчету всех невыполненных операций, располагающихся перед ней. Операции расположенные ниже при таком запуске не пересчитываются.

Процесс закрытия месяца состоит из последовательного выполнения этапов, указанных на форме. Нажатие на кнопку Выполнить операции последовательно выполняет указанные действия в автоматическом режиме. При автоматическом расчете блокируются все кнопки панели, появляется панель с выводом информации о том, какая операция рассчитывается.

Если установлен флажок Останавливаться по ошибке и если в результате расчете операции не был диагностирован статус Выполнено успешно, то работа автоматического расчета прекращается, в противном случае выполняется расчет других операций.

Для каждого этапа показывается его состояние в текущий момент времени:

  • Не требуется - состояние указывает на то, что выполнение этапа в закрываемом месяце не требуется.
  • Не выполнено - состояние указывает на то, что выполнение этапа необходимо, но в данный момент он еще не выполнялся, или предыдущая попытка его выполнения закончилась неудачей.
  • Выполнено с ошибками - состояние указывает на то, что данный этап выполнен частично.
  • Выполнено успешно - состояние указывает на то, что все действия, выполняемые в рамках этапа, выполнены успешно.

Для уменьшения количества отображаемой в форме информации можно скрыть операции, выполнение которых не требуется, и выполненные успешно операции. Соответствующие команды находятся в подменю Настройки.

Этапы на форме упорядочены по принципу взаимозависимости - для этапов расположенных ниже необходимо успешное выполнение этапов расположенных выше.

Предварительное закрытие месяца

При предварительном закрытим месяца выполняются только часть регламентных операции, необходимые для расчета себестоимости за указанную часть месяца и отражения результатов данного расчета:
- Актуализация движений по данным взаиморасчетов;
- Распределение затрат и расчет себестоимости – в части этапов:
- Распределение партий;
- Распределение дополнительных расходов по товарам. Выполнение данного этапа можно отключить;
- Расчет себестоимости;
- Формирование движений по регистрам;
- Отражение документов в регламентированном учете.

При предварительном закрытии месяца на этапе распределения партий выполняется подбор партий только для тех документов, для которых партия ранее не была подобрана. Для документов, для которых были подобраны партии при текущем расчете, выполняется корректировка стоимости списания номенклатуры и запись движений по регистрам. Таким образом, при каждом предварительном закрытии месяца обрабатываются только новые или измененные документы. Это позволяет обеспечить сохранение подобранных партий и рассчитанную предварительную стоимость.

Окончательное закрытие месяца

В режиме окончательного закрытия месяца при расчете себестоимости рассчитываются суммы отклонений окончательного расчета от результата предварительного расчета. Отклонения выделяются отдельными записями по регистрам с указанием режима закрытия месяца при котором были сформированы движения.
Это позволяет наглядно видеть отклонения от ранее рассчитанной предварительной стоимости.

Ручные операции

Все этапы в данной группе выполняются вручную.

Очистка неиспользуемых движений по регистрам учета себестоимости

При выполнении этапа анализируется наличие движений по регистрам учета себестоимости в периодах, в которых учет себестоимости был отключен. При наличии таких движений открывается форма обработки Очистка неиспользуемых движений по регистрам учета себестоимости, в которой можно запустить процедуру очистки движений.

Исправление дублей видов запасов

При выполнении этапа анализируется наличие видов запасов с одинаковыми значениями реквизитов. При их наличии открывается форма списка справочника Виды запасов, в которой можно запустить процедуру удаления дублей.

Исправление ошибок в исходных данных

При выполнении этапа производятся проверки корректности данных информационной базы (документов, регистров), относящихся к нескольким этапам закрытия месяца.

Исправление отрицательных остатков товаров организаций

При выполнении этапа анализируется присутствие отрицательных остатков товаров на конец рассчитываемого периода. При их наличии открывается помощник исправления отрицательных остатков, в котором возможно исправить данные остатки.

Распределение расходов будущих периодов

При выполнении этапа анализируется необходимость распределения расходов будущих периодов. Если такая необходимость существует, то открывается форма списка документов Распределение расходов будущих периодов, в которой можно создать необходимые документы распределения.

Оформление передач товаров

При выполнении этапа анализируется необходимость оформления документов Передач товаров между организациями. Если такая необходимость существует, то открывается форма рабочего места Оформление передач товаров между организациями, в которой можно оформить документы передачи товаров.

Оформление передач и возвратов товаров (по данным, введенным до версии 11.4)

В данном этапе контролируется необходимость оформления документов Передач товаров между организациями и Возвратов товаров между организациями по данным, которые были введены до перехода на версию 11.4. Если такая необходимость существует, то открывается форма рабочего места Оформление передач и возвратов товаров между организациями, в которой можно оформить документы передачи товаров.

Оформление выкупа товаров

В данном этапе контролируется необходимость оформления документов по ответственному хранению товаров. Если такая необходимость существует, то открывается форма рабочего места Оформление передач и выкупов, в которой можно оформить требуемые документы.

Оформление отчетов по комиссии между организациями

При выполнении этапа анализируется необходимость оформления документов Отчет по комиссии между организациями. Если такая необходимость существует, то открывается форма рабочего места Отчеты по комиссии между организациями, в которой можно оформить требуемые документы.

Оформление отчетов комитентам о списании

При выполнении этапа анализируется необходимость оформления документов Отчет комитентам о списании. Если такая необходимость существует, то открывается форма рабочего места Оформление отчетов комитенту о списании, в которой можно оформить требуемые документы.

Оформление счетов-фактур и таможенных документов

При выполнении этапа анализируется необходимость оформления операций по учету НДС. Если такая необходимость существует, то открывается форма рабочего места Помощник по учету НДС, в которой можно оформить требуемые документы.

Начисления по кредитам депозитам и выданным займам (вручную)

При выполнении этапа анализируется необходимость начисления. Если такая необходимость существует, то открывается форма рабочего места документа Начисления по кредитам и депозитам, в которой можно оформить требуемые документы.

Взаиморасчеты и переоценка валютных средств

Все этапы в данной группе выполняются автоматически.

Формирование движений по расчетам с партнерами и переоценка расчетов (для режима "офлайн")

Производит формирование отложенных движений по заданиям из регистров "Задания к распределению расчетов с клиентами" и "Задания к распределению расчетов с поставщиками" по регистрам расчетов с партнерами по расчетным документам.

Производит расчет курсовых разниц взаиморасчетов на конец месяца - остатки взаиморасчетов пересчитываются по курсу валюты на конец месяца, а если в регистрах взаиморасчетов остаток в валюте расчетов равен нулю, то списывает остатки по суммам регламентированного и управленческого учёта.

Рассчитывает построчные суммы документов в валютах учёта для вывода в унифицированные печатные формы и учёта в себестоимости.

Актуализация движений документов по данным взаиморасчетов (для режима "онлайн")

Производит расчет курсовых разниц взаиморасчетов на конец месяца - остатки взаиморасчетов пересчитываются по курсу валюты на конец месяца, а если в регистрах взаиморасчетов остаток в валюте расчетов "0", то списывает остатки по суммам регламентированного и управленческого учёта.

Рассчитывает построчные суммы документов в валютах учёта для вывода в унифицированные печатные формы и учёта в себестоимости.

Производит начисление процентных расходов дисконтирования и рассчитывает приведенные стоимости ТМЦ для использования в себестоимости.

Расчет процентных расходов дисконтирования

Исходя из остатков задолженности поставщикам и условий в договорах контрагентов формирует движения по списанию и сторно списания процентных расходов дисконтирования.
Движения фиксируются в документе Расчет процентных расходов дисконтирования.

Переоценка денежных средств и финансовых инструментов

Создаются документы Расчет курсовых разниц.

Формирование затрат

В данной группе производится распределение всех нематериальных затрат. Все этапы в данной группе выполняются автоматически.

Аннулирование подарочных сертификатов

Создаются документы Аннулирование подарочных сертификатов.

Расчет себестоимости

Все этапы в данной группе выполняются автоматически. После успешного выполнения данных этапов появляется рассчитанная себестоимость товаров.

Распределение затрат и расчет себестоимости

Выполняется актуализация движений регистров партионного учета и себестоимости, распределение доп. расходов, формируются отложенные движения по регистрам партий.

Создаются документы Расчет себестоимости товаров. Вариант расчета себестоимости (Средняя за месяц, ФИФО (взвешенная оценка), ФИФО (скользящая оценка)) берется из учетной политики.

Формирование финансового результата

Все этапы в данной группе выполняются автоматически.

Оформление документов распределения расходов

Создаются документы РАСПРЕДЕЛЕНИЕ РАСХОДОВ, в которых выполняется настройка распределения расходов, списываемых на финансовый результат, по правилам распределения расходов, заданных для статьи расходов. Если для статьи расходов не задано правило распределения, то необходимо в рабочем месте задать настройки распределения вручную.

Распределение расходов по направлениям деятельности

Выполняется распределение расходов по направлениям деятельности.

Распределение доходов по направлениям деятельности

Создаются документы Распределение доходов по направлениям деятельности

Примечание. Для периодов, в которых расходы со списанием на фин. рез. распределялись с помощью документа «Распределение доходов и расходов между направлениями деятельности», будет действовать следующее правило:
- новые документы распределения расходов будет необходимо оформить только в случае, если изменилась сумма ранее распределенного расхода и появился нераспределенный остаток;
- в документе «Распределение доходов по направлениям деятельности» будет доступна закладка «Расходы», в том случае, если в ТЧ Расходы присутствуют записи.

Регламентированный учет

Все этапы в данной группе выполняются автоматически.

Формирование записей книги покупок и продаж

Выполняется формирование движений в регистры накопления для записей книг покупок и продаж по приобретенным ценностям за период.
Записи в книгу покупок и продаж формируются только по тем счетам-фактурам, по которым есть задания.
